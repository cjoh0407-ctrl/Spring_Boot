### 래퍼 클래스
***

자바가 객체 지향 언어(OOP)라는 것을 기억하시나요?  
이론적으로 이는 자바의 모든 데이터 타입이 객체라는 것을 의미합니다.

하지만, 8개의 예외인 기본 데이터 타입(primitive data types)이 있다는 것을 알고 계실 겁니다.  
기본 타입은 원래 *C* 및 *C++* 프로그래머들이 언어에 친숙함을 느끼도록 하고  
어느 정도의 유연성과 효율성을 제공하기 위해 자바에 포함되었습니다.  
그 대가로 OOP 패러다임이 깨졌습니다.

자바를 사용하다 보면 기본 타입을 객체로 사용할 일이 생기는데,  
그런 경우 *래퍼 클래스(wrapper classes)*가 필요한 도구가 됩니다.

본질적으로, *래퍼 클래스*는 해당 기본 데이터 타입을 포함하도록 설계된  
생성자를 가지고 있습니다.

8개의 기본 데이터 타입이 있는 만큼, 8개의 *래퍼 클래스*가 있습니다.  
아래 표는 이를 요약한 것입니다.

| **기본 타입**       | **래퍼 클래스**    |
|--------------------|-------------------|
| boolean            | Boolean           |
| byte               | Byte              |
| short              | Short             |
| int                | **Integer**       |
| long               | Long              |
| float              | Float             |
| double             | Double            |
| char               | **Character**     |

래퍼 클래스는 해당 기본 타입의 첫 글자를 대문자로 쓰는 버전으로,  
특이하게도 *int*와 *char*는 예외이므로 주의해야 합니다.

기본 타입을 객체로 변환하는 과정을 '박싱(boxing)'이라고 부르며  
(객체를 정보를 담고 있는 상자에 비유한 것입니다),  
래퍼 클래스로부터 해당 기본 타입의 객체를 다시 기본 값으로 변환할 때는  
그 변수의 '언박싱(unboxing)'이라고 부릅니다.

대부분의 경우에는 박싱이나 언박싱을 신경 쓰지 않아도 됩니다.  
컴파일러가 대부분의 변환을 자동으로 처리하기 때문입니다.  
이것을 '오토박싱(autoboxing)' 또는 '오토언박싱(autounboxing)'이라고 합니다.
